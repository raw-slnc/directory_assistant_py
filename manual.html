概要
Overview

DirectoryAssistantPy とは

What is DirectoryAssistantPy?

ブラウザで動くローカルファイルナビゲーターです。起動するとローカルサーバーを開始し、ブラウザで現場フォルダの全体像を表示します。

A browser-based local file navigator. Launch the server with a double-click, and your browser displays the full folder structure of your project.

🌲 できること
🌲 What it does
  • フォルダをサイドバーのツリーで階層表示・検索
  • Browse your folder hierarchy in a sidebar tree with search
  • ファイルをダブルクリックして OS のデフォルトアプリで開く（Excel は Excel で）
  • Double-click any file to open it with the OS default app (Excel opens in Excel)
  • 右クリックからファイルマネージャーで表示（Explorer / Finder）
  • Right-click to reveal in file manager (Explorer / Finder)
  • パンくずリスト・前後ボタンによるブラウザライクなナビゲーション
  • Breadcrumb navigation and back/forward buttons
  • ブラウザタブを閉じると 30 秒後にサーバーが自動終了
  • Server shuts down automatically 30 seconds after the browser tab is closed
🚫 設計上できないこと
🚫 By design, it does NOT
  • ファイルの作成・移動・名前変更・削除
  • Create, move, rename, or delete files
  • ネットワーク越しの別端末からのアクセス
  • Serve files to other devices over the network
  • ブラウザ内でのファイル内容プレビュー
  • Preview file contents inside the browser
  • クラウド同期・ユーザー認証
  • Cloud sync or user authentication
📝 ファイルの編集は Excel・Word などの慣れたアプリで行います。このツールは「探して開く」だけを担当します。 📝 File editing stays in Excel, Word, and your usual apps. This tool handles finding and opening only.
事前準備
Prerequisites

Python 3 のインストール

Installing Python 3

このツールの実行には Python 3 が必要です。インストールは最初の一回だけです。

Python 3 is required to run this tool. You only need to install it once.

1
ランチャーをダブルクリック Double-click the launcher

Python が見つからない場合、自動でインストールガイド（PythonInstallGuide.html）がブラウザで開きます。

If Python is not found, the launcher automatically opens PythonInstallGuide.html in your browser.

2
ガイドに従ってインストール Follow the install guide

公式サイト（python.org）からインストーラーをダウンロードして実行します。約 5 分で完了します。

Download the installer from python.org and run it. Takes about 5 minutes.

3
再度ダブルクリックで起動完了 Double-click again — you're done

次回からはガイドなしで即起動します。

From now on it launches immediately without the guide.

⚠ Windows の注意: Python インストール時に 「Add Python to PATH」 にチェックを入れてください。入れないと認識されません。 ⚠ Windows: When installing Python, make sure to check "Add Python to PATH", or the launcher will not find it.
起動方法
How to Launch

OS 別の起動方法

Launching by OS

ランチャーを現場フォルダに置き、ダブルクリックするだけです。

Place the launcher in your project folder and double-click it.

🪟
Windows
DirectoryAssistantPy.bat
  • ファイルをダブルクリックします。
  • Double-click the file.
  • Python 未導入の場合はガイドが自動で開きます。
  • If Python is missing, the install guide opens automatically.
🍎
macOS
DirectoryAssistantPy.command
  • ファイルをダブルクリックします。
  • Double-click the file.
  • Python 未導入の場合はガイドが自動で開きます。
  • If Python is missing, the install guide opens automatically.
  • 初回はセキュリティ確認が表示される場合があります。
  • On first launch, macOS may show a security warning — click Open anyway.
⚠ 実行できない場合：
トラブルシューティング
を参照してください。 ⚠ If the file won't open: See Troubleshooting.
🐧
Linux
DirectoryAssistantPy.sh
  • ターミナルで ./DirectoryAssistantPy.sh を実行します。
  • Run ./DirectoryAssistantPy.sh in a terminal.
  • ファイルマネージャーからの実行も可能です（実行可能に設定が必要な場合あり）。
  • Can also be launched from the file manager if marked executable.
🌐 ブラウザが自動で開かない場合は、手動で http://localhost:8742 にアクセスしてください。 🌐 If the browser does not open automatically, navigate to http://localhost:8742 manually.

バージョン情報
Version Information

2 種類のバージョン番号

Two Version Numbers

version と DA_VERSION は役割が異なり、別管理されています。

version and DA_VERSION serve different purposes and are managed separately.

version
配布・リリース管理番号
Release / distribution number

配布セット全体（ランチャー 3 ファイル + マニュアル）の管理番号です。パッケージのバージョンを識別します。

Identifies the distribution package as a whole (all 3 launchers + manual). Used to track releases.

DA_VERSION
本体スクリプトの更新判定
Core script update check

ランチャーに埋め込まれた DirectoryAssistant.py のバージョンです。起動時に既存ファイルと比較し、古ければ自動で上書き更新します。

The version embedded in DirectoryAssistant.py inside each launcher. On startup, the launcher compares this value against any existing file and overwrites it if outdated.

💡 なぜ分けるのか: ランチャーのデザイン変更など「配布形態の更新」と、Python スクリプト本体の機能変更は独立して行われるため、バージョンを別管理することで柔軟な更新が可能になります。 💡 Why separate? Distribution changes (launcher design, docs) and core script updates happen independently. Keeping them separate allows either to be updated without bumping the other.
